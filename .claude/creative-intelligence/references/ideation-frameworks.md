# Creative Intelligence — Ideation Frameworks & Design Thinking Reference

Load when running a brainstorm session, facilitating ideation, or generating creative options.

---

## Design Thinking Process

```
1. EMPATHIZE  → Understand the user's world (interviews, observation, empathy maps)
2. DEFINE     → Frame the right problem (How Might We statement)
3. IDEATE     → Generate many options without judgment
4. PROTOTYPE  → Build the cheapest version that can be tested
5. TEST       → Learn from real users; iterate
```

**Key principle**: Separate divergent (generating ideas) from convergent (selecting ideas).
Never evaluate during diverge. Never generate during converge.

---

## How Might We (HMW) Statements

Converts a problem into an opportunity for design:

```
Template: "How might we [verb] [outcome] for [user] [context]?"

Too narrow: "How might we add a search bar to the dashboard?"
Too broad:  "How might we make users happy?"
Just right: "How might we help new users find their first insight within 5 minutes?"
```

**From insight to HMW:**
```
Insight: "Users give up when they can't find their data"
↓
HMW make data discovery feel effortless?
HMW reduce the steps to get from login to first insight?
HMW help users who don't know what to search for?
```

## Brainstorm Facilitation

### Rules (post on wall / share in call)
1. Defer judgment — no "yes, but" during ideate
2. Build on others' ideas — "yes, and..."
3. Go for quantity — 100 ideas > 10 polished ideas
4. Wild ideas welcome — the ridiculous sparks the practical
5. Stay focused on the HMW — one topic at a time
6. One conversation at a time — no side discussions

### Warm-up (5 min)
"List as many uses for a brick as possible in 2 minutes."
Loosens up associative thinking before the real problem.

### Diverge Techniques

**Brain dump** (5 min): Silently write one idea per sticky. No talking.

**Round robin**: Each person shares one idea, build on previous. No skipping.

**Worst possible idea**: "What's the WORST solution to this problem?"
Then reverse it → often surfaces good ideas.

**Analogous inspiration**: "How does a hotel solve this problem? A hospital? Netflix?"

## SCAMPER Technique

Systematic prompt-based ideation for improving an existing product/concept:

| Letter | Prompt | Example (applied to onboarding) |
|---|---|---|
| Substitute | What can we replace? | Replace video tutorial with interactive walkthrough |
| Combine | What can we merge? | Combine sign-up + first task into one flow |
| Adapt | What can we adapt from elsewhere? | Adapt Duolingo streaks for daily usage |
| Modify / Magnify | What if we made X bigger / smaller / faster? | Magnify first success moment — celebrate loudly |
| Put to other uses | What else could this be used for? | Onboarding flow as sales demo tool |
| Eliminate | What can we remove? | Remove all optional fields from sign-up |
| Reverse / Rearrange | What if we did it backwards? | User picks their goal before they sign up |

## Converge Techniques

### Dot voting
Each participant gets 5 stickers. Place stickers on the ideas you find most promising.
Top-voted ideas move to the next round.

### 2×2 Prioritization Matrix
```
         High Impact
              │
  Easy ───────┼─────── Hard
              │
         Low Impact

Plot ideas on the matrix. "High impact + Easy" → do first.
```

### "How would [X] solve this?"
Pick a company / character / expert known for a specific strength:
- "How would Apple design this?" (obsessed with simplicity)
- "How would Amazon solve this?" (obsessed with speed + scale)
- "How would a 10-year-old explain it?" (ruthlessly simple)

## Storytelling for Ideas (Pitching Creative Work)

### Situation-Complication-Resolution (McKinsey narrative)
```
Situation:     Set the scene — what's true today?
Complication:  What's the problem / tension / change that demands action?
Resolution:    What's the solution and why will it work?
```

### The Before/After Story
```
Before: [Vivid description of the painful current state]
After:  [Vivid description of the improved future state]
Bridge: [How this idea gets us there]
```

Always make the user (not the company) the hero of the story.

## Creative Block Breakers

When stuck, try one of these:
- **Random word**: Open dictionary, pick a random word → force a connection to the problem
- **Opposite day**: List all your assumptions → flip every one
- **Time travel**: "How would we solve this in 1990? In 2050?"
- **10× thinking**: "What if we had 10× the budget? 1/10th the budget?"
- **The expert room**: Imagine 5 experts from different fields. What would each say?
- **Walk away**: 10-minute walk, no phone → often produces the breakthrough
